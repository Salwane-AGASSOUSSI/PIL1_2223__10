RAPPORT FINAL PROJET INTEGRATEUR

Objectifs du projet

Le projet Intégrateur avait pour objectif de développer une application web pour la gestion des emplois du temps dans un établissement scolaire. Les principaux objectifs étaient :

  • Créer une interface conviviale pour les utilisateurs.
  • Permettre la création, la modification et la suppression des emplois du temps.
  • Implémenter un système de notifications pour les modifications.
  • Filtrer les données relatives aux emplois du temps.

Réalisations

Au cours du projet, les réalisations suivantes ont été accomplies :

  • Développement de l'application web utilisant les technologies HTML, CSS, JavaScript et Django (Python).
  • Mise en place d'une base de données MySQL pour stocker les informations des utilisateurs et des emplois du temps.
  • Intégration d'un système de notifications en temps réel.
  • Création de rapports statistiques à partir des données de l'application.

Résultats et bénéfices

Le projet Intégrateur a été livré dans les délais prévus et a atteint les objectifs fixés. Voici les principaux résultats bénéfiques obtenus :

  • L'apprentissage et l'appariation de Git et GitHub.
  • L'apprentissage et l'utilisation de Django pour la réalisation du projet.
  • Gestion du travail en groupe et des problèmes internes qui en découlent.

Manuel d'utilisation de l'application

L'administration ( un superuser est creer et les identifiants sont communiquer a l'administration pour que le coordonnateur prenne ce compte. ) :
Le super utilisateur a le droit de :

  • Créer les emplois de temps hebdomadaires.
  • Modifier les emplois du temps.
  • Ajouter d'autres administrateurs simples afin de bien gérer l'application.
  • Supprimer ces administrateurs en cas de problème.

Les étudiants :
Les étudiants, après avoir créé leurs comptes utilisateurs, doivent remplir un formulaire de demande d'invitation pour utiliser l'application. Ce formulaire sera envoyé au supers utilisateurs. Ces derniers peuvent accepter ou refuser la demande en fonction de l'appartenance ou non de chaque étudiant à leur école. Une fois la demande acceptée, les étudiants recevront un e-mail leur notifiant que la demande a été acceptée et qu'ils possèdent désormais un compte pour se connecter. En cas de refus, un e-mail sera également envoyé pour notifier aux étudiants de reprendre le formulaire ou de se rendre à l'administration physique de l'école pour les réclamations.

Faut noter que les 2 types d'users se connecte au meme endroit.
